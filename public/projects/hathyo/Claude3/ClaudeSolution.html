2.2 Nguyên tắc thiết kế (cập nhật Claude3)

  1. HH chỉ phát sinh từ giao dịch thực (order DELIVERED/COMPLETED) – không phải từ tuyển dụng.
  2. Vô hạn cấp: Mạng lưới phát triển L0 → L1 → L2 → ... → Ln, không giới hạn depth.
  3. 5 slot AC tính từ người mua ngược lên: L(n-1) nhận ac1=5%, L(n-2) nhận ac2=2%, ..., L(n-5) nhận ac5=0.5%. Tổng = 10%.
  4. alfco: Khi người mua ở cấp n<5 (không đủ 5 upline), phần pool dư alfco được hoàn vào ví ảo người mua.
  5. 1.000 Founders là nút gốc (L0), không có parent, được tạo bởi Admin.
  6. Không còn self-purchase bonus (khác Claude2): người tự mua không nhận thêm HH. Chỉ nhận alfco nếu n<5.
  7. Thuế TNCN 10% khấu trừ tại nguồn khi chi HH ≥ 2 triệu/lần. alfco miễn thuế.

3.1 Định nghĩa và Phân loại

Founders là 1.000 tài khoản đặc biệt được Admin tạo thủ công, là nút gốc L0 (depth_from_root = 0), không có parent. Họ có mã giới thiệu riêng để phát triển cây mạng lưới bên dưới.

3.2 Đặc quyền Founder

  • Mã định danh: HAT-F0001 → HAT-F1000, badge 🏆 FOUNDER trên mọi màn hình.
  • Nhận HH từ ngày 1: Nhận ac5=0.5% khi cháu ở L5 mua hàng; ac4, ac3 từ những cháu gần hơn trong đúng vị trí chuỗi.
  • Không giới hạn HH/tháng: Founder không bị cap tháng.
  • Không bị downgrade: Trừ khi Admin thu hồi (is_founder không tự mất).
  • Hỗ trợ ưu tiên: Dedicated support channel, onboarding riêng.

3.3 Mô phỏng tăng trưởng – Số Dunbar k=16

Theo lý thuyết Dunbar: mỗi người duy trì ~16 mối quan hệ có ảnh hưởng thực sự. Hệ thống thiết kế với k=16 (trung bình mỗi nút phát triển 16 nút trực tiếp).

4 Quy Tắc Nghiệp Vụ Chi Tiết MỚI

4.1 Tính toán commission_base

commission_base = order.subtotal − order.discount_amount -- KHÔNG bao gồm: phí vận chuyển, VAT -- Ví dụ: Đơn 1.200.000đ − voucher 200.000đ − ship 30.000đ -- commission_base = 1.000.000đ -- affiliate pool = 1.000.000 × 10% = 100.000đ

4.2 Quy tắc phân bổ AC – Công thức tổng quát

Người mua ở cấp n → alfc = min(n, 5) → Phân bổ ac1..ac(alfc) cho L(n-1)..L(n-alfc).
Nếu alfc < 5: alfco = 10% − Σ(aci, i=1..alfc) → hoàn vào ví ảo người mua.

6.3 Bảng affiliate_tree_closure – Closure Table (query O(1))

-- Closure table: lưu tất cả quan hệ ancestor-descendant -- Cho phép query 5 upline gần nhất trong O(1) không cần CTE đệ quy CREATE TABLE affiliate_tree_closure ( ancestor_id UUID NOT NULL REFERENCES affiliate_nodes(node_id), descendant_id UUID NOT NULL REFERENCES affiliate_nodes(node_id), depth INTEGER NOT NULL, -- 0=self, 1=parent, 2=grandparent... (không giới hạn) PRIMARY KEY (ancestor_id, descendant_id) ); CREATE INDEX idx_closure_desc ON affiliate_tree_closure(descendant_id, depth); CREATE INDEX idx_closure_anc ON affiliate_tree_closure(ancestor_id, depth); -- Khi thêm node B (child) với parent A: INSERT INTO affiliate_tree_closure(ancestor_id, descendant_id, depth) SELECT :b, :b, 0 -- self UNION ALL SELECT ancestor_id, :b, depth + 1 FROM affiliate_tree_closure WHERE descendant_id = :a; -- ancestors của A -- Query 5 upline gần nhất của buyer X (cho commission engine): SELECT n.*, c.depth FROM affiliate_tree_closure c JOIN affiliate_nodes n ON n.node_id = c.ancestor_id WHERE c.descendant_id = :buyerNodeId AND c.depth BETWEEN 1 AND 5 ORDER BY c.depth ASC;

8 Commission Engine – Thuật Toán & Triển Khai MỚI

8.1 Tổng quan luồng xử lý

Commission Engine chạy ASYNC sau khi OrderDeliveredEvent được publish. Không block checkout flow. Spring @Async + @Transactional đảm bảo atomicity.

8.2 Pseudocode Java Spring Boot

@Service public class CommissionEngineService { // AC rates: index 0 = ac1 (5%), index 4 = ac5 (0.5%) private static final double[] AC_RATES = {5.0, 2.0, 1.5, 1.0, 0.5}; private static final double SUM_AC = 10.0; @EventListener @Async("commissionPool") public void onOrderDelivered(OrderDeliveredEvent e) { processCommission(e.getOrderId()); } @Transactional public void processCommission(UUID orderId) { Order order = orderRepo.findById(orderId).orElseThrow(); if (order.isTestOrder()) return; long base = order.getSubtotal() - order.getDiscountAmount(); if (base <= 0) return; Optional<AffiliateNode> buyerNodeOpt = nodeRepo.findByUserId(order.getBuyerUserId()); if (buyerNodeOpt.isEmpty()) { platformPool.add(base * SUM_AC / 100, orderId, "NO_NODE"); return; } AffiliateNode buyerNode = buyerNodeOpt.get(); int n = buyerNode.getDepthFromRoot(); // buyer's level int alfc = Math.min(n, 5); // Query up to 5 closest ancestors via closure table List<UplineInfo> ancestors = closureRepo.findAncestors(buyerNode.getNodeId(), 1, alfc); List<CommissionConfig> acConfigs = configService.getActiveConfigs(); // ac1..ac5 long totalPaid = 0; for (int i = 0; i < alfc; i++) { if (i >= ancestors.size()) break; UplineInfo upline = ancestors.get(i); if (upline.getStatus() != ACTIVE) { logSkip(orderId, upline, "ACCOUNT_SUSPENDED"); continue; } double rate = acConfigs.get(i).getRatePercent(); // AC_RATES[i] long gross = (long)(base * rate / 100); long pit = gross >= 2_000_000 ? (long)(gross * 0.10) : 0; long net = gross - pit; saveCommissionTx(orderId, buyerNode, upline.getNode(), i+1, base, gross, pit, net); walletService.addPending(upline.getNode(), net); totalPaid += gross; } // alfco – ví ảo người mua long poolTotal = (long)(base * SUM_AC / 100); long alfco = poolTotal - totalPaid; if (alfco > 0) { double alfcoPct = SUM_AC - sumAcRates(alfc); saveAlfcoPending(orderId, buyerNode, n, alfc, alfcoPct, alfco); walletService.addAlfcoPending(buyerNode, alfco); } // Phần còn lại do skip → platform pool // (đã xử lý trong từng skip case) } private double sumAcRates(int alfc) { double sum = 0; for (int i = 0; i < alfc; i++) sum += AC_RATES[i]; return sum; } // Khi order COMPLETED: approve HH + alfco @EventListener @Async("commissionPool") public void onOrderCompleted(OrderCompletedEvent e) { commissionRepo.approveByOrder(e.getOrderId()); // PENDING → APPROVED alfcoRepo.approveByOrder(e.getOrderId()); // PENDING → AVAILABLE walletService.movePendingToAvailable(e.getOrderId()); } }

9.4 Lưu ý bảo mật

  • Referral code: dùng base62 nanoid (20 ký tự) để tránh brute-force đoán mã.
  • Rate limiting: API validate referral code giới hạn 10 request/phút/IP.
  • AC calculation: server-side hoàn toàn, không tin vào client.
  • Payout OTP: Xác nhận qua ZaloOA/SMS trước khi tạo payout request.
  • Dual-control: Payout >10 triệu cần 2 admin duyệt.
  • Audit log: Mọi thay đổi AC config, Founder create, payout approve đều có audit trail.
  • Anti-circular: Closure table constraint tự động ngăn A→B→A (circular referral).
  • alfco fraud: Phát hiện khi alfco_total_received >50% tổng doanh thu của node → alert admin.
  • Depth bombing: cfgMaxDepth mặc định 20 trong commission engine để tránh vô tận loop.

10 Phụ Lục – Từ Điển Nghiệp Vụ

Thuật ngữĐịnh nghĩa
MLAMMulti-Level Affiliate Marketing – Tiếp Thị Liên Kết Đa Tầng. Hệ thống phân phối hoa hồng qua mạng lưới giới thiệu nhiều cấp.
Founder (L0)1.000 nút gốc được Admin tạo; depth_from_root = 0; không có parent; tham gia nhận HH từ chuỗi 5 cấp bên dưới khi đứng đúng vị trí.
Cấp n (Ln)Vị trí trong cây: L0=Founder, L1 do Founder giới thiệu, L2 do L1 giới thiệu, ... Ln không giới hạn.
ac1..ac55 slot hoa hồng tính từ người mua ngược lên: ac1=5% (gần nhất), ac5=0.5% (xa nhất, tối đa 5 bậc). Tổng = 10%.
alfcAffiliate Levels for Commission = min(n, 5). Số cấp upline thực sự nhận HH khi người mua ở cấp n.
alfcoAffiliate Level Commission Offset = 10% − Σ(aci, i=1..alfc). Phần pool dư khi n<5; hoàn vào ví ảo người mua sau khi đơn COMPLETED.
Ví ảo (alfco wallet)Số dư alfco của người mua. Dùng để giảm giá đơn hàng kế tiếp. Không rút được thành tiền mặt. Miễn thuế TNCN.
commission_basesubtotal − discount_amount (không tính ship, VAT). Dùng để tính affiliate pool.
affiliate pool10% của commission_base. Tổng phân bổ cho tối đa 5 upline + alfco.
platform_poolPhần HH bị skip (không có upline, account suspended) quay về sàn.
PITPersonal Income Tax – Thuế Thu Nhập Cá Nhân. 10% khấu trừ tại nguồn khi chi HH ≥ 2 triệu/lần.
Closure TableBảng lưu tất cả quan hệ ancestor-descendant để query upline chain trong O(1), không cần CTE đệ quy mỗi lần.
pending_balanceHH đang chờ order COMPLETED; chưa rút được.
available_balanceHH đã APPROVED; sẵn sàng rút (tối thiểu 200K).
GDTTổng Cục Thuế – cổng nộp khai thuế etax.gdt.gov.vn.
KBNNKho Bạc Nhà Nước – nơi nộp tiền thuế hàng tháng.
